## Контекст и почему Вариант A

### Что было найдено
1. **P0-1:** PIN-коды `0000` (admin) и `1111` (host) хардкожены в [src/data/initialData.ts:29-33](../src/data/initialData.ts#L29-L33), импортируются в [src/components/AdminView.tsx:5](../src/components/AdminView.tsx#L5), сравниваются клиент-сайд. После `vite build` попадают в `dist/assets/*.js` plain-text. Маршрут `/admin` достижим напрямую через [App.tsx:35](../src/App.tsx#L35) и [public/admin/index.html](../public/admin/index.html).
2. **P2-8 (бывший P0-2):** все 13 admin-write вызовов (POST/PATCH/DELETE) идут через анонимный PostgREST без JWT. После Garden phase 18 (2026-05-04) `web_anon` имеет только `GRANT SELECT`. Все admin-writes падают 401.
3. **ARCH_CHECK:** EventsAdmin дублируется Garden, остальные (Notebooks/Questions/Cities) — уникальны в meetings, но **физически не работают с 2026-05-04**.

### Verification 2026-05-05 (Chrome smoke-runner)
`POST /notebooks` без JWT возвращает **503/401 + PostgREST `42501 permission denied for table notebooks`**. Запись не создаётся. Подтверждает: phase 18 работает корректно, дыры в Garden нет, **админка meetings физически сломана**.

### Логика решения
Защищать сломанный функционал PIN'ом / Basic Auth / IP-allowlist'ом — бессмысленно: даже если защитить, кнопки всё равно вернут 401. Восстановление требует архитектурного решения (Telegram-Auth → JWT, либо переезд в Garden), это **отдельный таск с другим scope** — см. [NB_RESTORE_PLAN.md](NB_RESTORE_PLAN.md).

В рамках текущего таска SEC-PINS: **полностью удалить мёртвый код**, чтобы:
- убрать PIN-коды из бандла (P0-1 закрывается);
- убрать поверхность атаки `/admin`;
- убрать ~1720 строк мёртвого кода;
- убрать destructive-методы из `lib/postgrest.ts` (фронт после правки — read-only);
- сократить бандл (минус `html-to-image`, `file-saver`, `mapbox-gl`, часть Radix);
- бесплатно закрыть 1 npm vuln (`protocol-buffers-schema` через `mapbox-gl`).
